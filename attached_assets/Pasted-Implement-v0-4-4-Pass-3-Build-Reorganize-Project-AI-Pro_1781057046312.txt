Goal:
Add an AI prompt mode/template that helps the user build or reorganize a complex project using LifeGrid JSON patches.

This mode should support use cases like:
- learning to fly and getting certified
- starting a complex hobby
- preparing for credentialing
- planning a move
- reorganizing an existing project around competing priorities
- pausing/deprioritizing an existing project and shifting tasks/events safely

Do not:
- add direct AI API integration
- add backend/auth/cloud sync
- change localStorage key
- add auto-apply
- add a full scheduling optimizer
- add external calendar import
- weaken transformation safety rules
- let candidateDeletes apply
- allow title/date/time delete matching
- silently delete events or tasks

5. Prompt instructions should tell AI:

You are helping build or reorganize a LifeGrid project.

When creating a new project:
- Use projects.add.
- Add project notes explaining purpose, assumptions, and scope.
- Create tasks.add for milestones and next actions.
- Create flexible events.add for planned work blocks.
- Use eventKind: flexible-work-block for flexible schedule blocks.
- Use reviewItems.add for uncertain requirements, assumptions, dependencies, or items needing user review.
- Use dueDateType and triageStatus where useful.
- Use projectId consistently across tasks.
- Link events to related tasks when possible using linkedTaskIds / linkedEventIds.

When reorganizing an existing project:
- Use stable IDs for tasks.update and events.update.
- Move or reprioritize flexible items when appropriate.
- Use tasks.update to mark items backlog, waiting, blocked, ready, or done as appropriate.
- Use events.update for flexible-work-block, reminder, or placeholder items only when moving schedule blocks.
- Do not move/delete fixed-appointment, shift, travel, protected-time, day-type, or unknown eventKind items.
- Use mergeIntoDayType or convertTimedBlockToTask only when appropriate and only with stable IDs.
- Use candidateDeletes only as review-only suggestions.
- Put ambiguous issues into reviewItems.add or warnings.
- Do not delete any event or task unless it is clearly safe, stable-ID-based, and low-risk.
- Prefer reviewItems.add over risky destructive edits.

Output requirements:
- Return LifeGrid JSON patch only.
- No prose outside JSON.
- Use current patch v2 format plus supported review/transformation proposal sections.
- Use stable IDs for update/delete/transform proposals.
- Use reviewItems.add for uncertain findings.
- candidateDeletes are review-only and never auto-applied.
