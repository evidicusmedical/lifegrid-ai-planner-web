Fix v0.4.4 direct event delete preview so protected deletes are not shown as safe changes.

Current issue:
When importing an AI patch with events.delete containing a protected event ID, the apply logic correctly blocks deletion, but the AI import preview still shows:
- "1 safe change ready to apply"
- "Remove event: <id>"

This is misleading. Protected event deletes should be visible as blocked/review-needed, not safe.

Current safety rule:
Only these eventKind values may be deleted automatically:
- flexible-work-block
- reminder
- placeholder

These must be blocked from AI direct delete:
- fixed-appointment
- shift
- travel
- day-type
- protected-time
- missing/unknown eventKind

Required behavior:

   - Before showing events.delete as safe removals, check the existing event by ID.
   - If the event is deletable under the safe-delete rule, show it under safe changes.

   - Label clearly:
     "Blocked delete: <event title>"
     "This event is <eventKind or unknown> and will not be deleted automatically."
